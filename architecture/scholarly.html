Abstract

Vinyl recommender system is an web application which help users create and share music playlist and besides that giving the possibility to receive music recomandations based on user feedback

1. Introduction

2. Internal Data Structures/Models

2.1 Overview

We use internal data model to manage and organise the data within the application, enabling efficient storage, updates and retrieval of the store data

2.2 Data Model

User Model includes attributes such as username, email, password, and any other relevant user information.

Vinyl Model used to store informations about each vinyl. Can include attributes such as name, description, duration.

Playlist Model includes information about the owner and the name of the playlist.

User Preferences Module includes parsing and storing user preferences using NLP and convert preferences into SPARQL queries to filter relevant music records

2.3 Recommandation Service

This module will provide vinyl recomandations for an user based on multiple criterias such as user preferences and maybe considering user generated playlists or share playlist from his social network

2.4 User Interface

We will design an intuitive and user-friendly interface where users can input preferences, view recommendations, create and share playlists with friends. Also will provide a visually appealing display of recommended vinyl records with cover images, song information, and any other significant details

3. External Data Sources

3.1 Overview

We are gonna use external data sources such us external APIs and models so that users have a much better and fuller experience when they have to choose and find new songs to add to playlist and to significantly enhance the creativite of our music recommendation service.

3.2 Data Source

Music Ontology provides a vocabulary for publishing and linking a wide range of music-related data on the Web. Music Ontology data can be published by anyone as part of a web site or an API and linked with existing data, therefore creating a music-related web of data.

Last.fm is a music recommendation service that tracks user listening habits and provides personalized recommendations. Also allows to fetch user data including playlists and user recommendations. So we are gonna use it for receiving vinyl recomandations based on playlist that user build or share in common with his network.

Discogs is a comprehensive database of music releases, including vinyl records. We will use Discogs API that allows us to access and retrive informations such as album cover, genres, realease dates and tracklists.

3.3 Data Integration

4. Linked Data Principles

4.1 Explanation of Linked Data

Linked Data is a set of principles and best practices for publishing, connecting, and interlinking structured data on the web.The key principles of Linked Data are designed to make data more discoverable, usable, and interconnected.

Here are the core principles:
  • 1.Use Unique Identifiers (URIs)
  • 2.Use HTTP URIs
  • 3.Provide Useful Information with RDF
  • 4.Include Links to Other URIs
  • 5.Link to External Data

4.2 How the Solution Conforms

We are gonna use these principles in the following manners:

  • 1.We are gonna use proper and specific names to our pages routes which is gonna help for searching
  • 2.We will expose our application through HTTP witch will allow more recognision from external serivices and application.
  • 4.We will offer links to anothers services which will allow user to view more information about their favorite songs and artists.
  • 5.We will provide link to our used external data APIs so that user can find more details on how our recomandations system works.
